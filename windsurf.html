How to use Windsurf Rules

Persistent instructions for Windsurf / Devin Desktop Cascade. For workspace rules, create Markdown files as .devin/rules/*.md (preferred), with .windsurf/rules/*.md as a legacy fallback. Legacy .windsurfrules files are still read.

You can also manage rules from Cascade Customizations → Rules. Open any rule page and click Copy Rule to save into your workspace.

About Windsurf / Devin Desktop Rules

What are Windsurf / Devin Desktop rules?

Windsurf (now Devin Desktop) rules are persistent instructions that shape how Cascade writes, reviews, and refactors code across your workspace. They encode team conventions, stack-specific patterns, and guardrails so the AI stays consistent session to session — without repeating the same prompts every time.

Where should Windsurf rules be stored?

Save workspace rules as Markdown files at your repo root:

  • Preferred: .devin/rules/*.md
  • Legacy fallback: .windsurf/rules/*.md
  • Legacy single file: .windsurfrules (still read)

You can also create and edit rules from Cascade Customizations → Rules without touching the filesystem.

Rules vs Memories vs AGENTS.md

  • Rules — explicit, version-controlled instructions you author (e.g. coding standards, framework patterns). Best for team conventions you want in git.
  • Memories — facts Cascade learns from conversation and retains across sessions. Good for preferences; less precise than checked-in rules.
  • AGENTS.md — a project-level agent brief (often at repo root) describing architecture, commands, and workflow. Complements rules; use both when your project needs a high-level map plus detailed standards.

Best Windsurf rules by stack

  • Python / APIs: FastAPI Backend, Django Web, Python Web Scraping
  • AI / ML: Python AI/ML Engineering, LangChain RAG, Prompt Engineering
  • TypeScript / Node: TypeScript General, Node.js Express API, Zod Schema Validation
  • Mobile / systems: React Native, Go Microservice API
  • Data / quality: PostgreSQL Database, Vitest Unit Testing, ESLint & Prettier

FAQ

Where do Windsurf / Devin Desktop rules go?

Create Markdown files as .devin/rules/*.md (preferred) or legacy .windsurf/rules/*.md at your repo root. Legacy .windsurfrules files are still read. You can also manage rules from Cascade Customizations → Rules.

What is the difference between Rules, Memories, and AGENTS.md?

Rules are explicit instructions you write and commit (coding standards, stack patterns). Memories are facts Cascade retains from past conversations. AGENTS.md is a project brief describing architecture and workflow — use it alongside rules for broader context. See the guide above for details.

How do I use a rule from AI Agent Dock?

Browse the Windsurf directory, open a rule detail page, click Copy Rule, then save as a .md file under .devin/rules/ or paste into Cascade Customizations → Rules. Each rule has a permanent URL (e.g. /rules/fastapi-backend).

Can I edit rules after copying?

Yes. Rules are starting templates — remove sections you do not need, add project-specific conventions, and keep total length reasonable so you do not overrun the AI context window.

Are these rules free to use?

Yes. Most rules on this site are MIT-licensed unless noted otherwise on the rule page. Check the License line on each card or detail page.

How do I submit my own rule?

Visit Submit a Rule for guidelines, then send your rule by email.
